# Анализ опроса поведения пользователей мобильного приложения

Данные получены из публичного датасета с Kaggle “Mobile App User Behavior Dataset”. В нем хранятся данные опроса пользователей мобильного приложения.

**Цель проекта** провести EDA (устройство и платформа, демографический анализ, пользовательское поведение) и построить воронки поведения пользователей, чтобы узнать на каком этапе происходит самый большой отток клиентов, какие есть отличия поведения пользователей разных платформ и что нужно улучшать.  
___
# EDA

* Android доминирует (60–70%), iOS — второй по значимости сегмент
* В среднем: заходят раз в месяц, скачивают 2–5 приложений
* Пользователи выбирают осознанно: ищут, читают отзывы, сравнивают
* Основная цель — решить задачу или развлечься

## Портрет пользователя:

- Заходит в стор раз в месяц
- Скачивает 3–5 приложений
- Почти не платит
- Уходит при плохом UX или багах

> Пользователь платит не за приложение, а за:
>>удобство
>>>стабильность
>>>>отсутствие раздражения

# Воронка

<img width="588" height="498" alt="image" src="https://github.com/user-attachments/assets/5fc766d2-33f9-435b-b504-5cb310479efe" />

- Основной отток — ~15% пользователей не знают/не используют App Store → потеря на самом раннем этапе
- Высокая конверсия внутри стора, 80%+ пользователей: заходят скачивают → стор работает эффективно как канал
- Самый слабый этап — монетизация. Только ~52% платят → почти половина пользователей: остаются в free-сегменте
- “Узкое место” воронки. Главная проблема не в установках, а в конверсии в оплату


> Воронка сильная до установки
>> Основной рост — в монетизации и увеличении знания/доверия к сторам

→ Усиливать монетизацию

→ Работать с ранним этапом awareness

→ Не фокусироваться только на установках — там уже высокая конверсия
